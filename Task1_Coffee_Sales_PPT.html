Quality Assurance

Before modeling, we must eliminate noise that could skew weights in the Random Forest ensemble.

Statistical Filtering (Z-Score)

We use Z-scores to identify transactions whose values deviate significantly from the mean (> 3 standard deviations). This isolates structural anomalies.

Business Logic Filtering

We actively search for domain-specific anomalies, such as "training transactions" that have a monetary value of $0, ensuring our mean revenue calculations remain accurate.

anomaly_detection.pypython
import numpy as np from scipy import stats def filter_anomalies(data, column): # Identify values > 3 std dev from mean z_scores = np.abs(stats.zscore(data[column])) clean_data = data[z_scores < 3] anomalies = len(data) - len(clean_data) print(f"Isolated {anomalies} anomalies.") # Check for $0 training transactions zero_vals = clean_data[clean_data[column] == 0] if len(zero_vals) > 0: print(f"Found {len(zero_vals)} $0 transactions.") clean_data = clean_data[clean_data[column] > 0] return clean_data df = filter_anomalies(df, 'money')

The Feature Engineering Problem

The Problem with Hours

Extracting the "hour" from a datetime returns an integer from 0 to 23. To a machine learning model, the numerical distance between 23 (11 PM) and 0 (Midnight) appears large, even though they represent contiguous moments in time.

Business Logic Features

Beyond raw time, human scheduling operates on discrete social logic:

  • Is it the weekend?
  • Is it the morning rush?
  • Is it payday?
Cyclical Encoding Solution

We use Sine and Cosine transformations to map linear hours onto a continuous 24-hour circle.

sin(2 * π * hour / 24)

cos(2 * π * hour / 24)

feature_eng.pypython
def synthesize_time_features(data): # Extract raw hour data['hour'] = data['datetime'].dt.hour # 1. Cyclical Encoding (Trigonometric Transforms) data['h_sin'] = np.sin(2 * np.pi * data['hour'] / 24) data['h_cos'] = np.cos(2 * np.pi * data['hour'] / 24) # 2. Behavioral Flags mapped to integers constraints (0/1) # Weekends: 5=Saturday, 6=Sunday data['is_weekend'] = data['datetime'].dt.dayofweek.isin([5, 6]).astype(int) # Morning Rush: between 7 AM and 10 AM data['is_morning_rush'] = data['hour'].between(7, 10).astype(int) # Payday proxy: assuming spikes around 15th and end of month data['payday_proximity'] = data['datetime'].dt.day.isin([15, 30, 31]).astype(int) return data df = synthesize_time_features(df) print(df[['datetime', 'hour', 'h_sin', 'is_weekend']].head())

Time-Series Framework

Forecasting requires transitioning from individual event logs (receipts) to time-bounded aggregates ("Daily Units").

Autoregressive Lags

We engineer features by shifting past sales forward. A 7-day lag is statistically the most significant predictor because coffee consumption follows a strong weekly work-cycle pattern.

Exponential Smoothing

We calculate an Exponential Moving Average (EMA) to isolate the true demand trend, cutting through daily noise.

aggregation.pypython
# Aggregate to daily resolution daily = df.groupby(df['datetime'].dt.date).agg({ 'money': 'sum', 'coffee_name': 'count' # Proxy for total volume }).rename(columns={'coffee_name': 'sales'}) # Create 7 days of lags for i in range(1, 8): daily[f'lag_{i}'] = daily['sales'].shift(i) # Smooth the signal with 7-day EMA daily['ema_7'] = daily['sales'].ewm(span=7).mean() # Drop rows with NaNs introduced by lagging daily.dropna(inplace=True) print("Aggregated shape:", daily.shape)
